## autodpi چه می‌کند؟

اکثر ابزارهای دور زدن سانسور فقط **یک روش** دارند و شما را مجبور می‌کنند ده‌ها تنظیم را دستی امتحان کنید تا یکی جواب بدهد. autodpi برعکس عمل می‌کند:

1. **بررسی می‌کند (probe)** که سامانه‌ی فیلترینگ (DPI) شبکه‌ی شما چطور یک هدف را بلاک می‌کند.
2. **آزمایش و رتبه‌بندی می‌کند (tune)** که کدام روش دور زدن روی شبکه‌ی *شما* بهتر کار می‌کند.
3. **اجرا می‌کند (run)** یک واسط محلی که روش برنده را روی ترافیک شما اعمال می‌کند.

شما فقط کلاینت خودتان (Xray، V2Ray یا مرورگر) را به آدرس محلی این ابزار وصل می‌کنید و تمام.

---

## چرا به این ابزار نیاز داریم؟

سانسور DPI در هر ISP و هر کشور فرق دارد. کانفیگی که برای یک نفر کار می‌کند، برای دیگری شکست می‌خورد. سخت‌ترین بخش، **پیدا کردن روشی است که روی شبکه‌ی خود شما جواب بدهد** — و این دقیقاً همان کاری است که autodpi خودکار می‌کند.

---

## نکته‌ی مهم قبل از شروع

autodpi یک **ابزار خط فرمان (command-line)** است، نه برنامه‌ای که با دابل‌کلیک پنجره باز کند. اگر روی فایل دابل‌کلیک کنید، یک پنجره‌ی مشکی یک لحظه باز و بسته می‌شود — این ایراد نیست؛ ابزار منتظر دستور است.

برای استفاده باید آن را از داخل **ترمینال** اجرا کنید (در ویندوز: Git Bash یا CMD یا PowerShell؛ در مک/لینوکس: Terminal).

## نصب

### راه ۱ — دانلود باینری آماده (ساده‌ترین)

1. به صفحه‌ی Releases پروژه بروید:
   `https://github.com/Mehrschad/autodpi/releases`
2. فایل مناسب سیستم‌عاملتان را دانلود کنید:
   - **ویندوز:** فایلی که به `windows-msvc.exe` ختم می‌شود
   - **مک (اپل سیلیکون / M1/M2/M3):** `aarch64-apple-darwin`
   - **مک (اینتل):** `x86_64-apple-darwin`
   - **لینوکس:** `x86_64-unknown-linux-gnu`
3. فایل را در یک پوشه‌ی مشخص بگذارید (مثلاً `Downloads`).

> در مک و لینوکس باید به فایل اجازه‌ی اجرا بدهید:
> ```bash
> chmod +x ./autodpi-*
> ```

### راه ۲ — ساخت از روی سورس‌کد

اگر Rust نصب دارید:

```bash
git clone https://github.com/Mehrschad/autodpi.git
cd autodpi
cargo build --release
# فایل خروجی: ./target/release/autodpi
```

## سه دستور اصلی

| دستور   | کار                                                              |
|---------|------------------------------------------------------------------|
| `probe` | هر روش را یک‌بار روی یک هدف امتحان می‌کند و نتیجه را نشان می‌دهد    |
| `tune`  | همه روش‌ها را چند بار تکرار، رتبه‌بندی و بهترین را پیشنهاد می‌دهد   |
| `run`   | واسط محلی را با روش انتخاب‌شده از روی فایل کانفیگ اجرا می‌کند       |

برای دیدن راهنمای کامل خود ابزار:

```bash
./autodpi help
```

> در ویندوز اگر نام فایل کامل است، همان نام را بنویسید، مثلاً:
> ```bash
> ./autodpi-x86_64-pc-windows-msvc.exe help
> ```
> برای راحتی می‌توانید نام فایل را به `autodpi.exe` تغییر بدهید.

## راهنمای گام‌به‌گام استفاده‌ی واقعی

فرض می‌کنیم نام فایل را به `autodpi` (یا در ویندوز `autodpi.exe`) تغییر داده‌اید و در ترمینال داخل پوشه‌ی آن هستید.

### گام ۱ — آی‌پی هدف را پیدا کنید

این ابزار به‌جای نام دامنه به **آی‌پی** نیاز دارد. آی‌پی سرور خودتان را دارید؛ اگر می‌خواهید یک سایت را تست کنید، آی‌پی‌اش را پیدا کنید:

```bash
# مک / لینوکس / Git Bash
nslookup www.example.com
```

### گام ۲ — بررسی اولیه (probe)

ببینید کدام روش‌ها از سد DPI رد می‌شوند:

```bash
./autodpi probe --sni www.example.com --ip 93.184.216.34
```

خروجی چیزی شبیه این است:

```
strategy                      verdict
--------------------------------------
direct                       reset (DPI)
split-sni                            PASS  (12 ms)
split-1                              PASS  (9 ms)
multisplit-4                         PASS  (8 ms)
```

- `PASS` یعنی آن روش از سد رد شد ✅
- `reset (DPI)` یعنی فیلترینگ اتصال را قطع کرد
- `timeout` یعنی پاسخی نیامد

### گام ۳ — رتبه‌بندی و پیدا کردن بهترین (tune)

```bash
./autodpi tune --sni www.example.com --ip 93.184.216.34 --rounds 5
```

این دستور هر روش را ۵ بار تکرار می‌کند و در پایان بهترین را پیشنهاد می‌دهد:

```
recommended strategy: "split-sni"

add this to a listener in config.json:
  "strategy": "split-sni"
```

### گام ۴ — ساخت فایل config.json

یک فایل به نام `config.json` کنار برنامه بسازید (نمونه‌اش در ادامه آمده) و نام روش برنده را در آن بگذارید.

### گام ۵ — اجرای واسط (run)

```bash
./autodpi run --config config.json -v
```

حالا ابزار روی آدرس محلی (مثلاً `127.0.0.1:40443`) گوش می‌دهد. کلاینت خودتان را به همین آدرس وصل کنید.

---

## فایل config.json

```json
{
  "listeners": [
    {
      "listen": "127.0.0.1:40443",
      "connect": "104.18.4.130:443",
      "sni": "www.example.com",
      "strategy": "split-sni"
    }
  ]
}
```

| فیلد       | توضیح                                                                 |
|------------|------------------------------------------------------------------------|
| `listen`   | آدرس محلی که ابزار روی آن گوش می‌دهد (کلاینت را به این وصل کنید)         |
| `connect`  | آدرس سرور بالادست به‌صورت `ip:port` (باید آی‌پی باشد، نه نام دامنه)      |
| `sni`      | نام دامنه‌ای که موقع tune استفاده می‌شود (اختیاری)                       |
| `strategy` | نام روش انتخاب‌شده (اختیاری؛ پیش‌فرض `split-sni` است)                    |

می‌توانید چند `listener` داشته باشید — هر کدام به یک سرور وصل می‌شود.

## استفاده همراه Xray / V2Ray

به‌جای اینکه کلاینت VLESS/VMess را مستقیم به سرور واقعی وصل کنید، آن را به آدرس محلی autodpi وصل کنید:

1. در `config.json` ابزار، فیلد `connect` را روی **آی‌پی و پورت سرور واقعی** خود بگذارید.
2. autodpi را اجرا کنید (دستور `run`).
3. در کلاینت Xray/V2Ray، آدرس سرور را به `127.0.0.1` و پورت را به همان پورت `listen` (مثلاً `40443`) تغییر دهید.

autodpi کار دور زدن DPI را پشت صحنه انجام می‌دهد؛ دست‌دادن واقعی TLS کلاینت شما بدون دستکاری از داخل آن عبور می‌کند. **autodpi هیچ‌وقت ترافیک شما را رمزگشایی نمی‌کند.**

## جدول روش‌ها (استراتژی‌ها)

همه‌ی این روش‌ها فقط در سطح کاربر (userspace) کار می‌کنند — بدون نیاز به دسترسی root/admin.

| نام                 | روش                                                       |
|---------------------|-----------------------------------------------------------|
| `direct`            | بدون دستکاری (مبنا برای مقایسه)                            |
| `split-sni`         | برش جریان دقیقاً داخل نام دامنه‌ی SNI                       |
| `split-sni-slow`    | همان، با کمی تأخیر بین تکه‌ها                              |
| `split-1`           | برش بعد از اولین بایت                                      |
| `split-5`           | برش بعد از هدر رکورد TLS                                   |
| `multisplit-4`      | شکستن ClientHello به ۴ تکه‌ی کوچک                          |
| `multisplit-8-slow` | ۸ تکه با تأخیر (در برابر DPI که بسته‌ها را بازچینی می‌کند) |

## رفع اشکال‌های رایج

**با دابل‌کلیک هیچی اجرا نمی‌شود / پنجره سریع بسته می‌شود.**
این طبیعی است. ابزار را باید از داخل ترمینال با یک دستور (مثل `help`) اجرا کنید.

**`command not found` یا `... is not recognized`.**
مطمئن شوید داخل همان پوشه‌ی فایل هستید و در مک/لینوکس/Git Bash از `./` قبل از نام فایل استفاده می‌کنید.

**در ویندوز SmartScreen هشدار می‌دهد.**
چون فایل امضای دیجیتال تجاری ندارد. روی «More info» و سپس «Run anyway» بزنید (اگر به منبع فایل اطمینان دارید).

**همه‌ی روش‌ها `reset` یا `timeout` می‌شوند.**
ممکن است بلاک در سطح آی‌پی باشد یا به بک‌اند raw-socket نیاز باشد (در نقشه‌ی راه پروژه است و هنوز پیاده نشده). آی‌پی دیگری از همان سرویس را امتحان کنید.

**در مک «cannot be opened because the developer cannot be verified».**
یک‌بار این را بزنید:
```bash
xattr -d com.apple.quarantine ./autodpi
```

## محدودیت‌ها و واقعیت‌ها

- دور زدن DPI یک بازی موش‌وگربه است؛ **هیچ روشی همیشگی نیست**. هر وقت کار نکرد، دوباره `tune` بزنید.
- این ابزار فقط **ترافیک خروجی خودِ شما** را دستکاری می‌کند. به سرور مقصد حمله یا آسیب نمی‌زند.
- روش‌های فعلی (سطح کاربر) بخش بزرگی از DPI را دور می‌زنند، اما نه همه را. بلاک در سطح آی‌پی و برخی DPIهای پیشرفته به بک‌اند raw-socket نیاز دارند که هنوز در دست توسعه است.
